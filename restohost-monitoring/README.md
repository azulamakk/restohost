# RestoHost AI Monitor

Monitoring system for RestoHost's AI voice assistant. Analyzes phone calls from two restaurants, detects assistant failures, identifies patterns, and presents the results in an interactive dashboard.

## Requirements

- Node.js v20+
- npm v10+
- Anthropic API key (optional — see demo mode below)

## Setup

```bash
git clone <repo-url>
cd restohost-monitoring
npm install
```

## Environment variables

Copy the example file and fill in your API key:

```bash
cp .env.example .env
```

Edit `.env`:

```env
ANTHROPIC_API_KEY=sk-ant-...   # Your Anthropic API key
MOCK_LLM=false                  # Set to "true" for demo mode (no cost, no key needed)
```

> **Demo mode:** If you don't have an API key or want to test without cost, use `MOCK_LLM=true`. The classifier will fall back to local heuristics. Results are less precise for ambiguous cases, but the full pipeline runs fine.

## How to run

### Step 1 — Run the pipeline

```bash
npm run pipeline
```

Runs the full analysis and writes JSON files to `/output`. Takes ~1–2 min in LLM mode (300 calls × ~$0.025 total), or ~10 s in mock mode.

### Step 2 — Start the dashboard

```bash
npm run dashboard
```

Starts the API at `http://localhost:3001` and the React dashboard at [http://localhost:5173](http://localhost:5173).

### Run both in one command

```bash
npm run all
```

Runs the pipeline first, then opens the dashboard.

### Tests

```bash
npm test
```

## Pipeline output files

| File | Contents |
|---|---|
| `descriptive_stats.json` | Full descriptive stats: volume, duration, language, sentiment, repeat callers, etc. |
| `classified_calls.json` | All 300 calls with error classification, severity, and proposed fix |
| `tickets.json` | Only the calls with errors — formatted as actionable tickets |
| `patterns.json` | Temporal patterns, restaurant comparison, top 4 improvement opportunities |
| `forecast.json` | Next-week volume forecast with confidence level and disclaimer |
| `sentiment_analysis.json` | Per-call sentiment scores, emotion labels, motive groups, and mood↔outcome correlation |

## API endpoints

The Express server (`npm run api`) exposes:

| Endpoint | File served |
|---|---|
| `GET /api/stats` | `descriptive_stats.json` |
| `GET /api/calls` | `classified_calls.json` |
| `GET /api/tickets` | `tickets.json` |
| `GET /api/patterns` | `patterns.json` |
| `GET /api/forecast` | `forecast.json` |
| `GET /api/sentiment` | `sentiment_analysis.json` |

## API cost estimate

Using claude-haiku-4-5 with prompt caching enabled:
- **~$0.025 USD** per full pipeline run (300 calls)
- Prompt caching saves ~60% on the system prompt

Use `MOCK_LLM=true` to skip all API calls entirely.

## Dataset

`data/calls_dataset.csv` contains 300 anonymized real calls from March 2026:
- **TF Oakwood** — 150 calls, Spanish AI assistant, delivery and takeout focus
- **RT Buckhead** — 150 calls, English AI assistant, sit-down with reservations and takeout

The file is encoded in latin-1 (not UTF-8). The loader handles this automatically.
